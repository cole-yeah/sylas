## 代码规范

团队协作，规范的代码风格很有必要

### Github 提交规范

Header 部分只有一行，包括三个字段：type（必需）、scope（可选）和 subject（必需）
type(scope): subject, 如:
常用 type 如以下所示：

- feat: 新功能
- fix: 修复 bug
- docs: 文档
- style: 格式
- refactor: 重构（不是新增功能，也不是修改 bug 的代码改动）
- test: 增加测试功能
- chore: 构建过程或辅助工具的变动
- build: 打包
- ci
- perf: 性能优化
- revert
